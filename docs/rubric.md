# Final Project: 100-Point Checklist

## 1. Pre-Production & Submission
- [ ] **Planning:** Create ideas, an outline (wireframe), and storyboard sketches that are well-executed and turned in on time .
- [ ] **Approval:** Ensure the storyboard and wireframe are approved .
- [x] **Uniqueness:** Ensure work is unique (no copying classwork) and utilizes 75% of tags/properties discussed in class .
- [ ] **Submission:** Zip the project folder and upload it to Online Delhi .
- [ ] **Presentation:** Present the project during the class showcase to earn credit .

## 2. File Structure & Architecture
- [x] **File Count:** Create at least **five** interlinked HTML pages, **one** CSS page, and **one** JS page .
- [x] **Organization:** Organize pages in a clear, logical manner with appropriate page lengths .
- [x] **Validation:** Validate all code (HTML and CSS) to ensure no errors .

## 3. HTML & SEO Elements
- [x] **Meta Data:** Include a descriptive `<title>` tag and `<meta>` description for **every** HTML page .
- [x] **Tagging:** Use appropriate tags, ensuring they are nested and ended correctly with correct attributes .
- [x] **Table:** Include one table with at least three rows and three columns containing unique content .

## 4. Content & Writing
- [x] **Volume:** Include at least **250+ words** of content on at least **four** different pages .
- [x] **Originality:** Ensure content is well-researched, accurate, relevant, and created by you (adds value to what is available online) .
- [x] **Conventions:** Ensure text is free of spelling, punctuation, spacing, and grammar errors .
- [x] **Citations:** Cite all sources properly using MLA format (can be on a separate page or in comments) .

## 5. Navigation & Links
- [x] **Menu:** Provide a well-thought-out main menu on **all** pages (use sub-menus if appropriate) .
- [x] **Status Indicators:** Indicate the current location using titles, headings, and menu status (e.g., highlighting the "active" page link) .
- [x] **Internal Links:** Ensure links are obvious, integrated neatly into the writing style, and change appearance to indicate a "visited" state .
- [x] **External Links:** Include 3 discrete links to relevant outside sources .
    - [x] Ensure one of these links opens in a new window (target="_blank") .
- [x] **Contact:** Include Webmaster contact information with a `mailto` link on one page .

## 6. Design & CSS (Aesthetics)
*Note: The "Aesthetics" category alone is worth 30 points.*
- [x] **Visual Appeal:** Design must be exceptionally attractive, match the topic, and advance the purpose of the site .
- [x] **Browser Compatibility:** Website must display correctly in all modern browsers .
- [x] **Layout Consistency:** Layout must be very useful, well-organized, and consistently applied across pages .
- [x] **Formatting:** Use whitespace and alignment effectively to organize content .
- [x] **Responsiveness:** Correct use of responsive design for both mobile and desktop views .
- [x] **CSS Techniques:**
    - [x] **Background:** Use a non-white background on at least one page .
    - [x] **Float:** Use a multi-column layout utilizing the `float` property on at least **two** pages .
    - [x] **Grid:** Use the CSS Grid element to display content .
    - [x] **Alignment:** Use appropriate and varied alignments (Left, right, center, Top, bottom, justified) .
    - [x] **Box Model:** Appropriate use of Borders, Padding, and Margins .

## 7. Media & Graphics
- [x] **Quantity:** Include at least **five** graphic elements .
- [x] **Quality:** Ensure graphics are relevant, well-cropped, and sized/controlled via CSS .
- [x] **Accessibility (Alt Text):** Use well-written `alt` attributes on **all** graphics .
- [x] **Video:** Include at least one video file (must be hosted in your file folder, not a link to YouTube/Vimeo) .
- [x] **Audio:** Include at least one sound clip (must be hosted in your file folder) .
- [x] **Social Media:** Integrate one social media tool relevant to the topic (X/Twitter, YouTube, Facebook, RSS, etc.) .

## 8. JavaScript & Interaction
- [x] **Form Input:** Prompt for user input through a form component (checkbox, radio button, dropdown, etc.) .
- [x] **Form Output:** Use the user's input from the form to make a change on the page or trigger a popup .
- [x] **Event Listener:** Include an `onclick` statement that functions to create output .

## 9. Accessibility
- [x] **Typography:** Ensure font size and typeface are appropriate for headings and body text .
- [x] **Contrast:** Ensure all text is attractive and easy to read against the site background ..
